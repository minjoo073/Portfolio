A. Placeholder.tsx — 가장 단순한 접근

type: 'image' | 'video' | 'empty' prop + useState(false) failed 플래그. img는 onError → setFailed(true). video는 src prop 없으면 바로 Placeholder 렌더 — fetch 404 런타임 체크 불필요, src 없음 자체가 fallback 신호로 충분.

// img 래핑
<img
  src={src}
  onError={(e) => {
    e.currentTarget.onerror = null; // 무한루프 방지 — 아래 위험 참조
    setFailed(true);
  }}
  ...
/>

// video fallback — src 없으면 바로 placeholder
{!videoSrc ? (
  <PlaceholderDiv label="VIDEO — COMING" showPlayIcon />
) : (
  <video src={videoSrc} ... />
)}

SVG play icon — Lucide PlayCircle (stroke text-ink-muted, 24px). Lucide는 이미 설치된 아이콘 셋이므로 추가 라이브러리 없음.

B. ProjectCard.tsx — 가장 단순한 접근

// studyHref 없으면 클릭 불가 래퍼
const CardWrapper = studyHref
  ? ({ children }) => <a href={studyHref}>{children}</a>
  : ({ children }) => <div role="presentation" aria-hidden="true">{children}</div>;

// siteHref 없으면 disabled 버튼처럼
<a
  {...(siteHref ? { href: siteHref } : {})}
  aria-disabled={!siteHref}
  className={cn('...', !siteHref && 'opacity-40 cursor-not-allowed pointer-events-none')}
>
  {siteHref ? 'View Site' : 'View Site — SOON'}
</a>

C. Footer + About — placeholder 판별 핵심 결론

hello@parkminjoo.com 하드코딩 비교 제거. 대신 data/footer.ts에서 email: '' (빈 문자열) 또는 null. 컴포넌트는 !email falsy 체크만.

// data/footer.ts — CEO 실값 받기 전
export const footerData = {
  email: '',          // 실값 오면 'parkminjoo@xxx.com' 채우기만
  resume: { url: '' }, // 실값 오면 '/resume.pdf'
};

// Footer.tsx
{email && <tr><td><a href={`mailto:${email}`}>{email}</a></td></tr>}

<a
  {...(resume.url ? { href: resume.url, download: 'resume.pdf' } : {})}
  aria-disabled={!resume.url}
  className={cn('...', !resume.url && 'opacity-40 cursor-not-allowed pointer-events-none')}
>
  Resume
</a>

About: !statement ? null : <section>...</section>. JSX 조건 렌더 — 레이아웃 gap 자동 소멸.

CEO 받는 즉시 1:1 교체 구조: data/footer.ts의 email · resume.url 값을 빈 문자열→실값으로만 바꾸면 컴포넌트 수정 없이 자동 렌더 전환.

위험 · 롤백

  1. GSAP 셀렉터 미스 (B 조건부 판정 이유) — <a>→<div> 교체 시 GSAP가 a 또는 .project-card a 셀렉터로 archive 카드를 애니메이션 타겟으로 잡고 있으면 target miss. 교체 후 브라우저 콘솔에서 GSAP targets found: 0 경고 확인 필수. 방지책: GSAP 셀렉터를 data-card 속성으로 통일 (a·div 모두 붙임).
  2. img onError 무한루프 — onError 핸들러에서 e.currentTarget.onerror = null 먼저 세팅 안 하면 빈 src 재셋 시 onError 재발화. state failed 체크로 2중 방어.
  3. resume.url 존재 ≠ 파일 존재 — resume.url = '/resume.pdf' 채워도 /public/resume.pdf 파일이 없으면 다운로드 404. 빌드타임 체크 불가 — CEO가 파일 올릴 때 경로까지 함께 확정 필요. (강체크 검증 포인트.)

롤백: 3개 파일 독립 변경. 빌드 실패 시 파일 단위 git restore. data/footer.ts 원래 placeholder 값은 주석으로 남겨둠.

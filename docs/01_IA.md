## 1. 사이트 구조 (Site Map)

```
PORTFOLIO (root)
│
├── / (Main / Home)
│   ├── 00  Intro              ← Browser Frame이 PORTFOLIO 타이포 위 굴절
│   ├── 01  About Me
│   ├── 02  Web Projects        (placeholder × 7)
│   ├── 03  Mobile Projects     (placeholder × 2)
│   ├── 04  Content & Marketing (종이 펄럭 인터랙션 유지)
│   ├── 05  Visual Works        (Horizontal Gallery)
│   ├── 06  Transition Scene    ← Browser Frame 회수 → 엔딩 시퀀스
│   └── 07  Footer              (Email · GitHub · Notion · Resume · Contact)
│
└── /projects (Project Detail · Long Scroll Page)
    ├── #project01  ← Web Project 01 제작과정
    ├── #project02  ← Web Project 02 제작과정
    ├── #project03  ← Web Project 03 제작과정
    ├── #project04  ← Web Project 04 제작과정
    ├── #project05  ← Web Project 05 제작과정
    ├── #project06  ← Web Project 06 제작과정
    ├── #project07  ← Web Project 07 제작과정
    ├── #mobile01   ← Mobile Project 01 제작과정
    └── #mobile02   ← Mobile Project 02 제작과정
```

- **메인은 SPA형 1페이지**, 프로젝트 디테일은 **별도 1페이지 + 앵커**로 분리.
- 분리한 이유: 메인 페이지는 "캔버스 위 떠다니는 브라우저 프레임"의 미장센이고, 제작과정은 HTML/CSS/JS/GSAP 학습 자료라 톤이 다름. 한 페이지에 섞으면 미장센이 무너짐.
- 두 페이지를 묶는 장치: **Browser Frame이라는 메타 오브제** — 메인에서 PORTFOLIO 위를 떠다니던 프레임이 Transition에서 회수돼 엔딩으로 수렴.

## 2. URL 구조

| URL | 역할 | 진입 경로 |
|---|---|---|
| `/` | Main · 캔버스 | 최초 진입 |
| `/#about` | About 섹션 점프 | (선택) 네비게이션 |
| `/#work` | Web Projects 진입 | (선택) 네비게이션 |
| `/projects` | 제작과정 첫 프로젝트로 진입 | 메인의 Project 01 클릭 |
| `/projects#projectN` | N번째 제작과정으로 점프 후 자연 스크롤 | 메인의 Project N 카드 클릭 |
| `/projects#mobileN` | 모바일 N번째 제작과정 | 메인의 Mobile N 카드 클릭 |

**핵심 UX 규칙**:
1. 클릭은 **앵커 점프**로 처리 — 페이지 전환 트랜지션 없음 (RayRayLab 톤 유지)
2. 앵커 점프 후 **계속 스크롤하면 다음 프로젝트로 자연 연결** — 사용자가 "01만 보고 끝"이 아니라 "여정을 따라가는" 느낌
3. 제작과정 페이지에는 **얇은 좌측 인덱스** (Project 01~07, Mobile 01~02) — 점프 보조

## 4. 섹션 흐름 논리 (왜 이 순서인가)

```
[Intro]                  → 디자이너로서 나의 메타포 제시 (Browser Frame)
   ↓ "내가 누군가?"
[About Me]               → 나의 경험·스킬을 사실로 설명
   ↓ "무엇을 만들었나?"
[Web Projects ×7]        → 가장 깊은 호흡, 메인 작업물
   ↓ "다른 매체에서는?"
[Mobile Projects ×2]     → 디바이스 컨텍스트 전환, 리듬 변주
   ↓ "디자인 외에는?"
[Content & Marketing]    → 출시·운영·마케팅 경험으로 폭 확장
   ↓ "비주얼 결과물은?"
[Visual Works]           → 수평 갤러리로 시각적 마무리
   ↓ "이 여정의 끝은?"
[Transition Scene]       → Browser Frame 회수 → 이름·역할·연락 노출
   ↓
[Footer]                 → 실제 연락 채널
```

**리듬 설계 의도**:
- Intro·About은 **느리고 정적** (정체성 확립)
- Web Projects는 **가장 길고 농밀** (포트폴리오의 핵심)
- Mobile은 **짧은 변주** (리듬 환기)
- Content & Marketing은 **종이 펄럭**이라는 단일 모션으로 차별화
- Visual Works는 **수평 전환**으로 리듬 완전 환기 (수직 피로 해소)
- Transition은 **밀도 최고** (엔딩의 카타르시스)
- Footer는 **정적 정보** (실용)

수직 → 수직 → 수직 → 수직 → **수평(환기)** → 수직(엔딩) 의 흐름이 핵심.

## 5. 프로젝트 탐색 메커니즘 (Project Discovery UX)

### 5.1 메인 페이지의 카드

각 프로젝트는 메인의 02·03 섹션에서 다음 요소로 구성:

```
[Meta]         날짜 · 카테고리 (작게, 좌상단)
[Title]        Project 01  (placeholder · 대형 타이포)
[Visual]       Placeholder 영역 (이미지 자리)
[Hover State]  커서 → "View" / 타이틀 우측 이동 / Visual scale 1.02
[Click]        → /projects#projectN 으로 앵커 점프
```

### 5.2 제작과정 페이지의 연속 흐름

```
/projects 진입
   ↓ (앵커가 #projectN이면 해당 위치로 점프)
[Project N 제작과정]
   - 의도 / 문제 정의
   - UX 리서치
   - IA / Wireframe
   - 인터랙션 / 모션
   - 최종 결과
   ↓ (자연 스크롤)
[Project N+1 제작과정]
   ↓
[Project N+2 ...]
```

**핵심 UX**:
- 사용자는 **한 프로젝트에 들어왔지만 다음 프로젝트로 자연스럽게 넘어감** → 작업물 간 비교/맥락이 자연 형성
- 좌측에 **얇은 인덱스 바** (sticky) — Project 01~07, Mobile 01~02 — 현재 위치 하이라이트
- 각 프로젝트 끝에 **"Next: Project N+1 →"** 푸터 (자연 스크롤 유도)
- 마지막 Mobile 02 끝에서 **"Back to Main ↑"** → `/` (Transition Scene 직전으로 점프)

### 5.3 진입/이탈 모션 (개요)

| 트리거 | 모션 |
|---|---|
| 메인에서 카드 클릭 | 화면 전체가 흰색으로 짧게 페이드 (200ms) → `/projects#projectN` 진입, 해당 섹션이 viewport에 들어와 있는 상태로 시작 |
| `/projects`에서 메인으로 복귀 | 좌상단 "← Back" 또는 마지막 푸터 — 같은 페이드 인버스 |

> 페이지 전환을 **드라마틱하게 만들지 않음** — quiet luxury 톤에 어긋남. 짧고 깔끔하게.

## 6. 글로벌 요소 (모든 화면 상시 존재)

| 요소 | 위치 | 역할 |
|---|---|---|
| **Navigation** | 상단 고정 (fixed) | 좌: `PARK MINJOO` · 우: `WORK` / `CONTACT` (간소화) |
| **Custom Cursor** | 전역 | 기본 12px 원, hover 시 48px + "View" 라벨, lerp 0.15 |
| **Browser Frame Objects** | 메인 페이지 상시 (3~5개) | PORTFOLIO 텍스트 위로 떠다니며 굴절. About~Visual Works 구간은 **밀도 낮추고**, Transition에서 다시 결집 |
| **Smooth Scroll (Lenis)** | 전역 | duration 1.2 · easing exponential |
| **Section Label** | 각 섹션 좌상단 | `(About Me)`, `(Web Projects)` 등 — RayRayLab의 `(Who we are)` 패턴 차용 |

**Browser Frame의 라이프사이클** (IA 차원의 큰 그림):
```
Intro      : 3~5개 떠다님 (가장 활발)
About      : 1~2개 잔향
Web/Mobile : 거의 없음 (작업물에 집중)
Content    : 1개 (종이 모션 위로 한 번 지나감 정도)
Visual     : 0~1개
Transition : 다시 모두 회수 → 중앙에 집결 → 엔딩 시퀀스
```
→ Browser Frame은 단순 장식이 아니라 **사이트 전체를 묶는 서사적 장치**.
